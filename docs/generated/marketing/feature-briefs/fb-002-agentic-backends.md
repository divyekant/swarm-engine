# Agentic Backends

## One-Liner

Give your AI agents real-world capabilities — file I/O, shell commands, and code execution — with plug-and-play agentic backends.

## What It Is

Standard AI agents can think and plan, but they can't actually do things. Agentic backends connect your workflow nodes to full execution platforms like Claude Code and Codex, letting agents read files, write code, run commands, and interact with real systems.

## Who It's For

- **Primary:** Engineering teams building code generation or automation workflows
- **Secondary:** DevOps teams automating infrastructure tasks

## Problem

Most AI orchestration tools limit agents to text generation. When you need agents that can actually execute code, modify files, or interact with systems, you're stuck building custom integrations.

## Key Benefits

- **Agents that do, not just think** — file I/O, shell, code execution
- **Mix freely** — combine thinkers (LLM) and doers (agentic) in the same workflow
- **Optional and modular** — install only the SDKs you need
- **Unified events** — same monitoring and cost tracking regardless of backend type
- **Bring your own** — custom agentic backends via a simple adapter interface

## How It Works

Add an agentic provider to your engine, mark specific nodes as agentic, and they automatically get access to real tools. Costs and events flow through the same system as standard agents.
